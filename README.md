# 🚗 CarWash - Premium Booking Platform

A modern, full-stack application ecosystem built for Car Wash centers and their customers. Features a stunning **Dark Theme Glassmorphism** mobile app for users and a lightning-fast Web Dashboard for center owners to manage their bookings.

---

## 📱 Mobile App (User Facing)

Built with **Flutter** for Android/iOS, the mobile application provides a premium, intuitive "Midnight Aurora" aesthetic for users to find and book car wash services near them.

### Features
* **Authentication**: Secure Login/Signup using Firebase Auth.
* **Geolocation Integration**: Automatically detects the user's location to sort and display the nearest car wash centers.
* **Dynamic Search & Filtering**: Zomato-style search UI with dynamic filtering by popular services (e.g., Detailing, Ceramic Coating).
* **Live Bookings**: Users can reserve specific time slots dynamically generated by the center's operating hours.
* **Real-time State Management**: Powered by Flutter `Provider` to ensure seamless, jank-free tab switching and booking updates.
* **Dark Glassmorphism UI**: Pitch-black backgrounds paired with elegant, frosted-glass cards.

### Tech Stack
* **Framework**: Flutter
* **Backend**: Firebase Authentication & Cloud Firestore
* **Key Packages**: `provider`, `geolocator`, `geocoding`, `url_launcher`, `google_fonts`

---

## 💻 Web Admin Dashboard (Partner Facing)

A standalone web application built specifically for Car Wash owners to manage their presence on the platform.

### Features
* **Partner Onboarding**: Owners can register and instantly add their Car Wash centers to the global database.
* **Location Auto-fill**: Instant coordinate generation using the browser's native Geolocation API.
* **Automated Slot Generation**: Automatically builds 1-hour booking slots mathematically based on the exact open and close times provided by the owner.
* **Live Booking Management**: View incoming appointments across all owned centers.
* **Status Updates**: 1-click "Mark Completed" buttons that securely update the user's mobile app in real-time.

### Tech Stack
* **Framework**: Vite + Vanilla JS (No overhead, lightning fast)
* **Styling**: Vanilla CSS with custom Glassmorphism tokens
* **Backend**: Firebase JS SDK (Auth + Firestore)

## 🚀 Getting Started

### Prerequisites
* Flutter SDK (`flutter doctor` should pass)
* Node.js & npm (for the web dashboard)
* Firebase Project Configuration (`google-services.json` inside `android/app`)

### 1. Running the Mobile App
1. Clone the repository.
2. Ensure you have your `google-services.json` placed in `android/app/`.
3. Fetch dependencies:
   \`\`\`bash
   flutter pub get
   \`\`\`
4. Run the app on a connected device or emulator:
   \`\`\`bash
   flutter run
   \`\`\`

### 2. Running the Admin Dashboard
The dashboard uses the same Firebase instance as the mobile app.
1. Navigate to the `admin_web` directory:
   \`\`\`bash
   cd admin_web
   \`\`\`
2. Install Javascript dependencies:
   \`\`\`bash
   npm install
   \`\`\`
3. Start the Vite development server:
   \`\`\`bash
   npm run dev
   \`\`\`
4. Open your browser to `http://localhost:5173`.

## 🔒 Security & Database
This project utilizes complex **Firestore Security Rules** to ensure:
* Users can only read and write their **own** bookings.
* Partners (Admins) can execute **Collection Group Queries** to read and complete bookings linked to their specific centers.
* *Note: Requires a Single-Field Exemption in Firestore Indexes on `bookings.centerId` for the admin dashboard queries to run.*
